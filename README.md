## Правила оформления

<a name="оформление--заголовок-главы"></a><a name="1.1"></a>

### Заголовок главы

- [1.1](#оформление--заголовок-главы) Заголовок главы оформляется как заголовок первого уровня. Номер и название разделяются точкой.

  ```md
  # Глава 663. Название
  ```

**[⬆ наверх](#содержание)**

<a name="оформление--тире"></a><a name="1.2"></a>

### Тире в тексте

- [1.2](#оформление--тире) Тире внутри повествовательного текста (перечисления, пояснения, вводные конструкции) оформляется двумя дефисами ` -- ` с пробелами. Pandoc преобразует их в короткое тире `–` при сборке.

  > Почему? Прямой символ `—` в исходнике труднее отличить от дефиса при редактировании. ` -- ` визуально явный и единообразный.

  **Плохо:**

  ```md
  западная часть — Республике Интис, а территория на востоке — Королевству Лоэн.
  ```

  **Хорошо:**

  ```md
  западная часть -- Республике Интис, а территория на востоке -- Королевству Лоэн.
  ```

**[⬆ наверх](#содержание)**

<a name="оформление--диалог"></a><a name="1.3"></a>

### Диалог

- [1.3](#оформление--диалог) Реплика без атрибуции начинается с `---` и занимает отдельный абзац.

  ```md
  --- Текст реплики.
  ```

- [1.4](#оформление--диалог) Если за репликой следует атрибуция или действие персонажа, они добавляются в ту же строку через ` --- `.

  > Почему? Это сохраняет реплику и её атрибуцию единым визуальным блоком, не разрывая читательский ритм.

  **Плохо:**

  ```md
  --- Текст реплики.

  Персонаж сделал что-то.
  ```

  **Хорошо:**

  ```md
  --- Текст реплики. --- Персонаж сделал что-то.
  ```

**[⬆ наверх](#содержание)**

<a name="оформление--мысли"></a><a name="1.5"></a>

### Мысли

- [1.5](#оформление--мысли) Внутренние мысли персонажа оформляются курсивом в кавычках-ёлочках.

  ```md
  *«Кровь?»*
  ```

- [1.6](#оформление--мысли) Атрибуция или действие после мысли добавляются в ту же строку через ` --- `.

  **Плохо:**

  ```md
  *«Текст мысли.»*

  Персонаж сделал что-то.
  ```

  **Хорошо:**

  ```md
  *«Текст мысли.»* --- Персонаж сделал что-то.
  ```

**[⬆ наверх](#содержание)**

<a name="оформление--звуки"></a><a name="1.7"></a>

### Звуки

- [1.7](#оформление--звуки) Звуковые эффекты и ономатопея оформляются символом `♫` без курсива.

  **Плохо:**

  ```md
  *Бах! Бах! Бах!*
  ```

  **Хорошо:**

  ```md
  ♫ Бах! ♫ Бах! ♫ Бах!
  ```

  Таблица переводов:

  | Оригинал | Перевод |
  |---|---|
  | Whoosh! | ♫ У-у-у! / ♫ Ф-ш-ш! |
  | Phew! | ♫ Фух! |
  | Bam! | ♫ Бах! |

<a name="оформление--артефакты"></a><a name="1.8"></a>

### Запечатанные артефакты

- [1.8](#оформление--артефакты) Идентификаторы Запечатанных артефактов оформляются как inline-код.

  **Плохо:**

  ```md
  Запечатанный Артефакт 0-08
  ```

  **Хорошо:**

  ```md
  Запечатанный Артефакт `0-08`
  ```

<a name="оформление--разделитель"></a><a name="1.9"></a>

### Разделитель сцен

- [1.9](#оформление--разделитель) Смена сцены или точки зрения обозначается центрированным символом `⁘` в блоке pandoc-разметки.

  ```md
  ::: {.centered}
  ⁘
  :::
  ```

<a name="оформление--клуб-таро"></a><a name="1.10"></a>

### Члены Клуба Таро

- [1.10](#оформление--клуб-таро) Обращения к членам Клуба Таро по роли карты заключаются в экранированные квадратные скобки `\[Название\]`. Экранирование обязательно, чтобы markdown не интерпретировал скобки как ссылку.

  **Плохо:**

  ```md
  [Повешенный] и [Мир] согласились на условия Деррика.
  ```

  **Хорошо:**

  ```md
  \[Повешенный\] и \[Мир\] согласились на условия Деррика.
  ```

- [1.11](#оформление--клуб-таро) Правило распространяется на все контексты: повествование, диалог и мысли.

  ```md
  *«\[Солнышко\], да ты простофиля!»* --- мысленно усмехнулся он.
  ```

<a name="оформление--пути"></a><a name="1.12"></a>

### Пути и Последовательности

- [1.12](#оформление--пути) Названия Путей пишутся с заглавной буквы в форме «Путь [Название]».

  **Плохо:**

  ```md
  путь смерти, путь гиганта
  ```

  **Хорошо:**

  ```md
  Путь Смерти, Путь Гиганта, Путь Бога Сражений
  ```

- [1.13](#оформление--пути) Названия ролей (ступеней) внутри Пути пишутся с заглавной буквы.

  ```md
  Привратник, Охотник на Демонов
  ```

- [1.14](#оформление--пути) Номер Последовательности указывается в форме `[N]-й Последовательности` перед названием Пути.

  **Плохо:**

  ```md
  Привратник 5 последовательности пути смерти
  ```

  **Хорошо:**

  ```md
  Привратник 5-й Последовательности Пути Смерти
  ```

  Полный пример:

  ```md
  Привратник 5-й Последовательности Пути Смерти может не только обычным образом перейти на следующую Последовательность своего Пути, но и выбрать Охотника на Демонов 4-й Последовательности Пути Гиганта, то есть Пути Бога Сражений.
  ```

## Полезные команды

Сборка epub по томам:

```sh
pandoc -i book/01-клоун.md -o build/epub/Повелитель\ тайн.\ Клоун.epub --split-level=2
pandoc -i book/02-безликий.md -o build/epub/Повелитель\ тайн.\ Безликий.epub --split-level=2
pandoc -i book/03-путешественник.md -o build/epub/Повелитель\ тайн.\ Путешественник.epub --split-level=2
```

### Прочие заметки

> Тома с 3 по 8 не отредактированы.

- `Sun domain` → сферы Солнца
- Район Шервуд
